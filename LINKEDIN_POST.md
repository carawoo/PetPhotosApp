## 📱 "그냥 찍어서 올리면 안 돼요?"

강아지가 너무 귀여워서 사진을 찍었는데...

❌ "이 사진은 3x3 배열에서 색감이 안 맞는데..."
❌ "프로필 피드가 지저분해 보이면 어쩌지..."
❌ "친구한테 사진 보여주다가 다른 사진 넘어가면 어떡하지..."

저도 그랬습니다.
그래서 만들었습니다. **Peto** 🐾

---

## 💡 핵심 아이디어

**"꾸미지 않아도 되는 반려동물 사진첩"**

- ✅ 지금 이 순간의 귀여움을 **바로** 기록
- ✅ 배열 고민 없이 **마음껏** 업로드
- ✅ 지인에게 보여줄 때 **오직 반려동물만** 노출
- ✅ 프라이빗한 일상은 안전하게 보호

---

## 🛠️ 2주간의 개발 여정

### 1️⃣ 기본 기능 구현
- **소셜 피드**: 사진 업로드, 좋아요, 댓글
- **반려동물 태그**: 우리집 강아지 이름으로 검색 가능
- **회원 시스템**: 간단한 닉네임 로그인

### 2️⃣ 사용자 경험 개선
**"비회원도 구경할 수 있어야 한다!"**
- 로그인 없이도 피드 구경 가능
- 단, 좋아요/댓글은 로그인 필요
- 비회원도 좋아요 누르면 하트 색 변경 (디바이스 저장)

### 3️⃣ 실용적인 기능들
**비밀번호 찾기 자동화**
- Web3Forms 서비스 연동
- 사용자가 비밀번호 찾기 신청하면
- 자동으로 관리자 이메일로 전송
- (별도 백엔드 서버 불필요!)

**검색 노출 최적화**
- Google Search Console 등록
- Naver Search Advisor 등록
- 메타 태그, 사이트맵 자동 생성

### 4️⃣ 브랜딩
- 발자국 아이콘 디자인 (분홍색 🐾)
- 로고와 파비콘 통일
- "Peto - 반려동물 사진첩" 타이틀

## 🎯 배운 점

### 기술적으로
- React Native Web로 크로스플랫폼 앱 개발
- Firebase로 빠른 프로토타입 제작
- Vercel로 무료 배포 (단, 하루 100회 제한 주의! 😅)

### 제품적으로
- **"덜어내는 것"이 더 어렵다**
  - 인스타그램의 복잡함을 빼고
  - 핵심 가치만 남기기

- **"사용자가 원하는 걸 만들되, 불편함을 해결하라"**
  - 비회원 접근: 친구들에게 쉽게 공유
  - 자동 이메일: 수동 작업 제거
